20220720

要求：

  1. 通过hostname命令更改本机名称为自己姓名的拼音；
  2. 新建一个db_web1数据库，新建一个user_web1用户，让user_web1拥有db_web1的所有权限；
  3. user_web1只能从本地和10.10.10.200上登录；

步骤：

  1. hostname yourname, hostname查看验证；
  2. vi /etc/hosts，将原来的hostname改为新的；
  3. mysql -uroot -p进入SQL语句编写；
  4. create database db_web1;创建数据库；
  5. create user 'user_web1'@'localhost';创建用户；
  6. grant all privileges on db_web1.* to 'user_web1'@'host' with grant option;赋权限；
  7. show grants for "user_web1"@"localhost";查看权限；
  8. create user 'user_web1'@'10.10.10.200';创建用户；
  9. vim /etc/mysql/mariadb.conf.d/50-server.cnf，将bind-address注释
  10. 防火墙ufw放行端口3306ufw allow 3306

公网IP，总结：计网太烂

SQL注入

手工注入：

  • 判断类型（字符or数值）、闭合符
  • 联合查询union
  • 报错注入mysql_error()
  • 布尔盲注
  • 延时盲注
  • GET/POST注入

工具辅助：sqlmap

概念和原理：攻击者通过注入sql语句片段导致后端执行预期外的SQL查询

危害：

  • 拖库：loadfile
  • 读写文件：outfile
  • 执行系统命令

另：

  • 网站数据被窃取
  • 网站页面被篡改
  • 服务器被攻击者控制

防护：

  • 对特殊字符进行转义：引号等
  • 正确使用预编译技术进行防护
  • 非必要不开启文件和命令执行权限
  • 进行库和用户的隔离

XSS

危害：

  • 窃取凭证cookie
  • 篡改页面
  • 构造XSS蠕虫

绕过方法：

  • 不同的js引入方式
  • 大小写绕过
  • 双写绕过
  • 字符串HTML实体化编码绕过，双引号中间的

防护：

  • 简单输入框：识别和转义特殊字符，eg.尖括号、引号、javascript中任意字符等
  • 复杂的富文本：使用成熟的前端富文本输入框组件，如uedior等
