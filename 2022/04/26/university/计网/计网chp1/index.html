一些概念

  • host主机
  • end system端系统
  • communication links通信链路
  • bandwidth带宽
  • packet switches分组交换机
    • router路由器
    • link-layer switch链路层交换机
  • Internet : network of networks
  • protocols协议
  • ISP，Internet Service Provider网络服务提供商
  • network edge网络边缘：hosts(client_server)
  • access networks接入网: wired/wireless ,连接网络边缘与边缘路由器的
    • 家庭接入
      • DSL：数字用户线
      • cable电缆
        • FDM：频分多路复用frequency division multiplexing
        • HFC
      • FTTP光纤到户
    • 企业接入enterprise
      • Ethernet以太网
      • wireless LANs=Wifi
    • 广域网无线接入
      • WAN
  • network core网络核心: interconnected routers
  • physical media
    • guided media电波沿固体媒体前行
      • twisted pair copper wire双绞铜线
      • coaxial cable同轴电缆
      • fiber optic cable光纤
    • unguided media电波在空气或外层空间传播
      • LAN-WiFi
      • wide-area
      • terrestrial microwave
      • satellite卫星无线电
  • 传输延迟packet transmission delay=L(bits)/R(bits/sec)
    • L: len(packet)
    • R: transmission rate
  • packet switching包交换，主机在应用层将message拆分成packets
  • circuit switching电路交换，预留资源
  • throughput吞吐量，发送方接收方之间的传输速率
  • Internet protocol stack协议栈
    • application: message
    • transport: segment
    • network: datagram
    • link: frame
    • physical: bit
  • ISO/OSI model: application, presentation, session,transport, network, link, physical
  • encapsulation封装
  • ICANN：管理域名和地址

Network Core

由端系统的分组交换机和链路构成，share

Packet-switching

每个分组都经过通信链路和分组交换机

store and forward

存储转发：在交换机转发之前必须收到整个分组

end-end delay

此处忽略传播时延，则端到端时延=2L/R。一半是交换机接收的时间(存储转发时延)，一半是destination接收的时间

queueing delay,loss

交换机存在一个输出队列（输出缓存），已到达的分组须在队列中等待，即有排队时延queue delay

若缓存区满，则出现丢包packet loss

routing&forwarding

circuit switching

dedicated，通常用于电话网络

复用

  • FDM频分复用
  • TDM时分复用

比较

  • 分组交换能更好地处理突然的数据
  • 分组交换可能面临巨大的拥塞

一个计算！

delay

  • $d_{proc}$: nodal processing
  • $d_{queue}$: queueing delay
  • $d_{trans}$: transmission delay=L/R
  • $d_{prop}$: propagation delay=d/s
    • d: len(physical link)
    • s: propagation speed传输速率

traffic intensity

  • 流量强度趋于1，排队时延趋于无穷
  • 设计系统时流量强度不能大于1

protocol stack

本书主讲TCP/IP协议栈，自顶而下

层次 协议 other
应用层5 HTTP, DNS, SMTP, POP3, IMAP, FTP, TELNET,
SSL, SSH→通过TCP实现
RIP, BGP		 host之间
传输层4 TCP, UDP, SCTP process之间
网络层3 IP, ICMP, ARP node(router)之间
链路层2 Ethernet, PPP node(switch)之间
物理层1

4,5是端到端，1,2,3是点到点

encapsulation

链路层交换机for下两层

路由器for下三层
