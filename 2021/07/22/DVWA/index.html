File Inclusion

文件包含漏洞

开发人员为了使代码更灵活,会将被包含的文件设置为变量,用来进行动态调用,从而导致客户端可以恶意调用一个恶意文件（恶意文件可通过文件上传漏洞实现上传）,造成文件包含漏洞。

文件包含vs目录遍历

  • 目录遍历：可以读取web目录以外的其他目录,根源在于对路径访问权限设置不严格，针对本系统。
  • 文件包含：利用函数来包含web目录以外的文件，分为本地包含和远程包含。

特征

  • ?page=test.php
  • ?home=test.html
  • ?file=content

low

没有过滤，可以通过?page=包含任意文件，如?page=http://127.0.0.1/phpinfo.php；

当包含一个不存在的文件时，页面显示报错，爆出网站的绝对路径。

首先会报错：reCAPTCHA API key missing from config file，意思是需要一个密钥，解决参照：DVWA报错： reCAPTCHA API key missing from config file error - 简书 (jianshu.com)

SQL Injection

目的：执行恶意SQL语句

手工注入的步骤

  1. 判断是否存在注入点，判断是字符型还是数字型
  2. 猜解SQL查询语句中的字段数：order by
  3. 确定回显位置：查看response
  4. 获取当前数据库、表、字段名、数据。

low

通过判断测试得知存在字符型注入（即有单引号闭合，构造SQL语句时要注意闭合单引号或有需要时注释掉后面的单引号），且有报错回显；

通过1' order by 2#得到有两个字段；

获取表：1' union select 1,group_concat(table_name) from information_schema.tables where table_schema=database()#

获取字段名：1' union select 1,group_concat(column_name) from information_schema.columns where table_name='users'#

获取数据：1' union select users,password from users#（密码使用了md5加密，可在https://www.cmd5.com/ 进行解密）

Weak Session IDs

弱会话IDs。用户在访问服务器时，服务器端会创建一个新的会话session，会话中保存用户的状态和相关信息，用于标识用户。

SessionID一般加密保存在cookie中，也可以保存在URL地址中。

low

通过burp可以直接看到session的值，观察可得到规律，每执行一次加一。获取了cookie值后，可绕过登录界面，直接输入用户界面的URL，同时用hackbar发送相应cookie值。
